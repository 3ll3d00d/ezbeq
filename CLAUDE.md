# ezbeq — Project Guide for Claude

## Project Overview

**ezbeq** is a full-stack web application that provides a browser and management interface for [BEQ (Bass EQ) filters](https://beqcatalogue.readthedocs.io/). It loads BEQ filters into audio DSP devices and supports:

- **MiniDSP** devices (2x4HD, 4x10, 10x10, DDRC-24, DDRC-88, HTx, SHD, 8x12CDSP, and others via minidsp-rs)
- **Monolith HTP-1** (Atmos processor)
- **JRiver Media Center**
- **Q-Sys** (via QRC protocol)
- **CamillaDSP**

## Architecture

```
ezbeq/           # Python backend package
  apis/          # Flask-RESTx REST endpoints
  *.py           # Device integrations and core modules
  ui/            # Built React frontend (served statically)
ui/              # React source code (MUI 5 + Vite)
  src/
    components/
    services/
tests/           # Pytest test suite
docs/            # MkDocs documentation
examples/        # Device configuration YAML examples
```

**Server architecture:** Flask (REST API at `/api`) + Twisted (static frontend + WebSocket) hybrid.

## Tech Stack

### Backend
- **Python 3.11+** required
- **Flask** + **Flask-RESTx** for REST API with automatic Swagger docs
- **Twisted** + **Autobahn** for async networking and WebSockets
- **PyYAML** for configuration
- **Poetry** for dependency management

### Frontend
- **React 18** + **Material-UI (MUI) 5**
- **Vite** as build tool
- **Vitest** for unit tests
- **Yarn 3.5.0** as package manager

## Development Setup

### Backend
```bash
poetry install          # Install Python dependencies
poetry run ezbeq        # Run the application (default port 8080)
```

### Frontend
```bash
cd ui
yarn install            # Install Node dependencies
yarn dev                # Dev server with hot reload
yarn build              # Build production assets to ezbeq/ui/
```

### Running Tests
```bash
# Backend
pytest                  # Run all tests
pytest --cov=ezbeq      # Run with coverage

# Frontend
cd ui
yarn test:unit          # Run Vitest unit tests
```

## Key Modules

| Module | Purpose |
|--------|---------|
| `ezbeq/main.py` | Flask app factory + Twisted server setup |
| `ezbeq/config.py` | Config loading from `~/.ezbeq/ezbeq.yml` |
| `ezbeq/catalogue.py` | BEQ catalogue management (HTTP caching, SQLite) |
| `ezbeq/device.py` | Abstract base classes: `Device`, `DeviceState`, `SlotState` |
| `ezbeq/minidsp.py` | MiniDSP integration (PEQ routing, biquad allocation, minidsp-rs WebSocket) |
| `ezbeq/camilladsp.py` | CamillaDSP integration (filter pipeline management) |
| `ezbeq/htp1.py` | Monolith HTP-1 integration |
| `ezbeq/jriver.py` | JRiver Media Center integration (MCWS API) |
| `ezbeq/qsys.py` | Q-Sys integration (QRC protocol) |
| `ezbeq/iir.py` | IIR/biquad filter utility functions |
| `ezbeq/apis/devices.py` | Device state/control API (v1, v2, v3 endpoints) |
| `ezbeq/apis/ws.py` | WebSocket server for real-time device updates |

## Coding Conventions

### Python
- **Python 3.11+** features are fine to use
- Follow existing code style in each module
- Device integrations follow the abstract base class pattern in `device.py` — new devices should implement `Device`, `DeviceState`, and `SlotState`
- API versioning is used (v1, v2, v3) — add new endpoints to appropriate version
- WebSocket events use the pattern established in `apis/ws.py`
- Configuration is parsed in `config.py` — add new device config parsing there

### Frontend (React/JSX)
- Functional components with hooks
- Material-UI components for UI elements
- Services layer (`ui/src/services/`) for API calls
- Keep device-specific UI in dedicated subdirectories (e.g., `components/minidsp/`)

### Tests
- **Backend:** Pytest with fixtures defined in `tests/conftest.py`
- Test catalogue data is at `tests/catalogue.json`
- Test configs at `tests/single.yaml`, `tests/multi.yaml`
- Use `pytest-httpserver` for mocking HTTP endpoints
- Mirror test structure from `test_minidsp_api.py` and `test_camilladsp_api.py` for new device integrations

## Configuration

The app reads from `~/.ezbeq/ezbeq.yml`. Example configs for all supported devices are in the `examples/` directory.

## API Documentation

Swagger UI is auto-generated by Flask-RESTx and available at `http://localhost:8080/api/` when running locally.

## Deployment

- **Standalone executable:** Built via `ezbeq.spec` (PyInstaller)
- **Systemd service:** Example in README for Raspberry Pi / Linux
- **ReadTheDocs:** Documentation hosted at https://ezbeq.readthedocs.io/
